How to read this document

Each feature is numbered in build order. Start at #1, finish at #30. Sequential build is mandatory — earlier features are foundations for later ones.

Every feature has three parts: visual mockup showing intended look and feel, specification describing behavior, and status badge.

Open decisions awaiting founder input are listed at the end. Items in that section block their related features — do not build until resolved.

Locked
Build at launch. Specification is final.
Phase 2
Build post-launch (after 90 days). Reserve technical hooks now.
Open
Awaiting founder decision (see end of document).
